# 📊 智慧健康日誌與風險評估系統 (Health Log & Risk Assessment System)

[cite_start]本專案為期末個人實戰挑戰——極速黑客松（Hackathon）之完整全端應用系統。專案針對 **題目 A：智慧健康日誌與風險評估系統** 進行全端開發，結合多層分支決策樹算法，實現即時健康風險監控 [cite: 91, 105]。

## 🚀 核心考核目標達成說明

### 01 資料流動的理解 (Frontend → API → DB)
[cite_start]本系統之資料流向清晰且完全打通 [cite: 99]：
1. [cite_start]**輸入表單送出**：前端正確呼叫 `POST /health-logs`，將使用者填寫的睡眠時數、步數、心情分數以 JSON 格式發送至後端 [cite: 119, 122, 187]。
2. [cite_start]**重新讀取列表**：資料成功寫入後，前端非同步機制（Fetch API）會立即動態重新呼叫 `GET /health-logs` 讀取最新歷史列表並即時呈現在畫面上 [cite: 119, 123, 187]。
3. **儲存機制**：後端配置記憶體虛擬資料庫 (dbMock) 進行高效能管理，確保黑客松環境下的資料一致性與高可用性。

### 02 模型的應用整合 (決策樹分類)
[cite_start]系統商務邏輯中無縫嵌入了 **多層分支決策樹模型 (Multi-layer Decision Tree)** [cite: 100, 102]：
- [cite_start]**第一層分支**：優先切分 `sleepHours` (睡眠時數) 特徵值 [cite: 102, 107]。
- [cite_start]**第二、三層分支**：依據睡眠結果，進一步下切判斷 `steps` (步數) 與 `moodScore` (心情分數) 的多層次交叉交叉組合 [cite: 102, 107]。
- [cite_start]系統能精準識別並處理「睡眠充足但完全不運動且心情極差」或「步數正常但心情差」等複雜的中間過渡情況，並以顏色徽章（紅/黃/綠）在前端完美呈現 [cite: 124, 189]。

### 03 與 AI 的協作效率 (Vibe Coding)
- [cite_start]靈活運用 AI 開發協作工具進行架構規劃、程式碼生成與即時錯誤除錯，落實「讓 AI 做苦工，人來做決策」之 Vibe Coding 核心精神 [cite: 58, 59, 89]。
- [cite_start]針對種子資料進行規律性安排（包含高、中、低風險組之極端值落差），並於本地端完成資料品質之自我檢查與驗證 [cite: 104, 114, 193]。
